# 사이트(프로젝트)별 운영 vs 전사 공통 표준 — 거버넌스

상단바의 **사이트 셀렉터**로 운영 범위를 선택한다. 사이트 범위 화면은 선택된 사이트의
데이터만 표시되고, 전사 공통 화면(사이드바 '전사 공통 표준' 그룹)은 사이트와 무관하게
하나의 표준을 공유한다.

## 1. 구분 기준

| 구분 | 모듈 | 소유/책임 | 비고 |
|------|------|----------|------|
| **사이트별** | 설비 마스터·생애주기 이력 | 각 사이트 | 이설 시에도 설비 이력은 연속 유지 |
| 사이트별 | PM 오더/실적, BM, 이슈, FDC 센서·알람 | 각 사이트 | 대시보드 KPI 도 사이트 범위 |
| 사이트별 | 비전 상태감시 포인트·순회 | 각 사이트 | 순회 영상은 사이트 단위로 수행 |
| 사이트별 | 투자 프로젝트·견적 | 각 사이트(프로젝트) | 업체 비교는 프로젝트 단위 |
| 사이트별 | 워크플로우 인스턴스 | 각 사이트 | 설비 미지정(모델 DR 등)은 공통 표시 |
| **전사 공통** | PM 표준 점검항목 (모델 단위) | 본사 표준 조직 | 사이트는 표준을 '적용'만 — 개정은 L&L 경유 |
| 전사 공통 | 워크플로우 템플릿 (7종 절차) | 본사 표준 조직 | |
| 전사 공통 | 라이프사이클 단계/프로세스 정의 | 본사 표준 조직 | |
| 전사 공통 | 지식 DB, 엔지니어링 계산 기준 | 본사 엔지니어링 | 출처·계수는 단일 소스 |
| **전사 공통 (풀)** | **Lesson & Learn** | 발생 사이트 → 전사 | 아래 흐름 참조 |
| (현 버전 공통) | 스페어파츠 재고 | — | Phase 3 에서 사이트별 창고로 분리 예정 |

## 2. L&L 흐름 — "사이트에서 발생, 전사로 확산"

```
사이트 A 에서 BM/이슈/워크플로우 → L&L 등록 (origin_site = A)
        │
        ▼ (등록 즉시 자동)
전사 공통 L&L 풀 등재 + 타 사이트 전원에 전파 레코드 생성 (NOTIFIED)
        │
        ▼ 각 사이트
사이트별 적용 상태 관리: 통보 → 검토중 → 적용완료 / 해당없음
        │
        ▼ (본사 표준 조직)
공통 PM 표준 점검항목으로 반영 (reflect-standard) → 전 사이트 차기 PM 부터 일괄 적용
        │
        ▼
차기 Design Review 데이터팩에 자동 포함 (미반영 L&L 강조)
```

핵심 원칙:
- **L&L 원장은 전사 공통 1개** — 사이트별 사본을 만들지 않는다 (이력 분산 방지)
- 사이트는 자기 적용 상태만 갱신, 내용 수정은 발생 사이트/본사만
- 표준 반영(PM 항목화)은 본사 표준 조직 권한 — 반영되면 전 사이트가 같은 표준을 본다
- 대시보드의 'L&L 법인 적용률'은 사이트 선택과 무관하게 전사 지표

## 3. 구현 상세

- 백엔드: 사이트 범위 엔드포인트에 `?site_id=` 파라미터 (Equipment.site_id 조인 필터) —
  dashboard / equipments / pm/orders / bm/reports / issues / fdc/sensors·alarms /
  vision-monitor/points·patrols / workflows
- 프론트: `SiteCtx` (상단바 셀렉터, localStorage 유지) → 사이트 범위 화면이 자동 재조회.
  전사 공통 화면에서는 셀렉터 대신 "전사 공통 — 사이트 무관" 배지 표시
- Phase 3 (다법인 확산) 에서 사이트별 사용자 권한(RBAC)·데이터 접근 통제로 강화
