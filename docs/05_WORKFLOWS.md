# 워크플로우 체계

설비 생애주기의 7개 표준 업무 절차를 워크플로우 엔진으로 관리한다.
모든 워크플로우는 단계(step)별 담당·메모·완료시각을 기록하고, 설비 지정 시 생애주기
이력에 자동 연동되며, 종료 시 L&L 자동 등록(전 법인 전파)이 가능하다.

| 유형 | 단계 수 | 핵심 포인트 |
|------|--------|------------|
| **DR** (설계 검토) | 8 | 듀티 검증 → **DR 데이터팩**(BM·PM NG·FDC·L&L 집계) → 안전율/수명 계산서 → 유지보수성 → 액션아이템 |
| **SETUP_STAB** (셋업 후 안정화) | 7 | 설치 정밀도 → 파라미터 확정(버전 등록) → 무부하/정격 시운전 → 안전장치 → 2주 안정화 감시 → 인수인계 |
| **ALARM_ACTION** (알람 조치) | 6 | 분류·심각도 → 현장 확인 → 원인 분석 → 조치 → 임계 재검토 → 재발 감시 |
| **BM_FLOW** (고장 정비) | 6 | 보고·LOTO → 응급복구 판단 → 5Why → 부품·재고 → 검증 운전 → 재발방지·L&L |
| **PM_FLOW** (예방 정비) | 6 | 오더 → LOTO → 표준 점검(비전 연계) → NG 처리 → 소모품 → 완료 보고 |
| **CONTROL_CHANGE** (제어 변경) | 6 | 요청 → 영향 분석 → 백업/롤백 → 적용 → 검증 → 파라미터 버전 갱신 |
| **SYS_ISSUE** (시스템 이슈) | 6 | 재현 조건 → 로그 분석 → 임시조치 → 벤더 에스컬레이션 → 패치 → 수평전개 |

## 데이터가 DR 로 환류되는 구조

```
운영 단계 데이터 축적                          차기 Design Review
─────────────────────                        ──────────────────
BM (원인·다운타임)        ─┐
PM 결과 (NG/CHECK 빈발)   ─┤
FDC 알람 (센서·분류 통계)  ─┼─► DR 데이터팩 ──► DR 워크플로우 2단계
L&L (미반영 항목 강조)     ─┤   (모델 단위        "과거 이력 데이터팩 검토"
Install Param 변경 빈도   ─┘    자동 집계)
지식 DB (카테고리 매칭)    ──►  근거 표준·문헌 자동 첨부
```

API: `GET /api/v1/workflows/dr-pack?model_id=` — 모델 단위로 위 데이터를 한 번에 집계.

## 단계 상태

`PENDING → IN_PROGRESS → DONE / NG / SKIP`. 모든 단계가 종결되어야 워크플로우 종료 가능.
NG 단계는 종료를 막지 않지만 결과 요약에 기록하고, 필요 시 L&L 로 등록한다.
