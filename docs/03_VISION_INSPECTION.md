# 비전 기반 PM/BM 설비 상태 측정 방법

PM 점검자가 스마트폰/현장 카메라로 촬영한 이미지를 업로드하면 서버가 자동으로 상태를
정량 측정하고 PM 판정기준과 비교하여 OK/NG 를 자동 판정한다. (운영 단계에서는 고정
카메라 + 주기 촬영으로 무인화)

## 1. 측정 종류 (kind)

| kind | 대상 예 | 알고리즘 | 출력 |
|------|---------|----------|------|
| `WEAR` | 호이스트 벨트, 휠 답면, 브레이크 패드 | 그레이스케일 임계 분할로 마모(어두운/밝은) 영역 비율 산출 | 마모율 % |
| `CORROSION` | 레일, 프레임, 볼트 | HSV 색공간에서 녹(적갈색) 픽셀 비율 | 부식률 % |
| `DIMENSION` | 와이어 직경, 휠 직경, 갭 | 기준 마커(체커보드/스케일 바)로 mm/px 캘리브레이션 후 엣지 간 거리 측정 | mm |
| `ALIGNMENT` | 레일 직진도, 컨베이어 롤러 정렬 | 엣지 직선 피팅 후 기울기/편차 | mm 또는 deg |

## 2. 처리 파이프라인

```
이미지 업로드(POST /api/v1/vision/inspect)
  → 전처리(리사이즈, 그레이스케일/HSV 변환)
  → 레시피 적용(threshold, ROI, mm_per_px 캘리브레이션)
  → 측정값 산출
  → PM 표준항목의 상·하한과 비교 → OK / NG / CHECK 자동 판정
  → VisionInspection 이력 저장, PM 결과에 자동 첨부
```

## 3. 캘리브레이션 (DIMENSION 측정)

- 촬영 시 피사체 옆에 **기준 스케일(예: 50mm 마커)** 을 함께 배치
- 레시피에 `ref_length_mm` 와 `ref_length_px`(또는 자동 마커 검출)를 등록 → `mm_per_px` 환산
- 동일 설비·항목은 촬영 지그를 표준화하여 레시피 재사용 (PM 표준항목의 `vision_recipe` 필드)

## 4. 판정 연계

- PM 표준항목에 `vision_capable=true` 인 항목은 PM 수행 화면에서 바로 이미지 업로드
- 측정값이 `lower_limit ~ upper_limit` 이내면 OK, 한계 ±10% 이내 접근 시 CHECK, 초과 시 NG
- NG 판정 시 BM 검토 권고 + 해당 파츠 스페어 재고 자동 표시

## 5. 고도화 로드맵

1. (현재) 고전 영상처리: 임계 분할·엣지 검출 — 레시피 기반, 설명 가능
2. 고정 카메라 + 정기 자동 촬영, 시계열 마모 트렌드 → 잔존수명(RUL) 추정
3. 딥러닝 결함 분류(크랙/박리/이물) — 사내 NG 이미지 축적 후 전이학습
4. FDC 센서(진동·전류)와 비전 결과 융합 판정
